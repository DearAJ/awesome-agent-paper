### W01（2025-12-28 至 2026-01-03）

- **#2 Youtu-LLM: Unlocking the Native Agentic Potential for Lightweight Large Language Models** [Tencent, 155↑] — arXiv 2512.24618
  - **动机**：sub-2B 区间长期被"蒸馏+对齐"方法主导，难以解锁原生 agentic 能力。Youtu-LLM 提出**从零预训练**路线，证明 1.96B 参数也能拿下 sub-2B 区间的 SOTA。
  - **架构**：dense MLA（Multi-head Latent Attention）+ 专门设计的 STEM 词表强化数学/代码 token + 128K 上下文窗口。**训练规模**：11T tokens，分 "Commonsense → STEM → Agent" 三阶段课程，每阶段对数据混合比和 sequence packing 做精细调度。
  - **核心创新**——**agentic mid-training**：在常规 pretraining 之后、SFT 之前插入一阶段，专门用合成的数学求解、代码编辑、多步工具调用轨迹训练，让模型在权重里内化"规划-反思-修正"循环，而不是靠 prompt 临时激发。
  - **结果**：在数学、code、tool-call benchmark 上均超过同等规模 Qwen3-1.7B 等基线；论文同时给出完整 data recipe、训练 infra 配置和 mid-training 数据合成 pipeline，是开源 sub-2B agent 模型领域近年最完整的一次公开。
- **#5 Let It Flow: Building ROME Model within Open Agentic Learning Ecosystem** [AGI Lab, 109↑] — arXiv 2512.24873
  - **定位**：不是单一模型，而是**端到端开源 agent 生产基础设施 ALE (Agentic Learning Ecosystem)**——填补"训练框架、环境管理、上层 agent CLI"三者割裂的空白。
  - **三大组件**：① **ROLL** 后训练框架，统一支持 SFT/DPO/GRPO/PPO，提供 agent rollout 的高吞吐流水线；② **ROCK** 沙盒环境管理器，可一键拉起 terminal/browser/code execution 多种容器并暴露统一 step API；③ **iFlow CLI** 上层 agent 框架，给最终用户。
  - **模型产出**——基于 ALE 训练的 ROME 模型，配套提出 **IPA (Interaction-Perceptive Agentic Policy Optimization)** 算法：将 credit assignment 从"逐 token"提升到"语义交互块"粒度（一次 tool 调用 + 观察 = 一块），显著改善长程 trajectory 的训练稳定性，避免格式 token 主导梯度。
  - **数据规模**：100 万+ agent trajectories，覆盖 50+ 任务类型。同时发布 **Terminal Bench Pro**——在原 Terminal Bench 200 任务基础上去除歧义、加强自动验证的改进基准。整个 ecosystem 完全开源（代码、权重、轨迹、环境镜像），是 2026 H1 开源 agent infra 的代表性工作。

### W02（2026-01-04 至 01-10）

- **#4 Youtu-Agent: Scaling Agent Productivity with Automated Generation and Hybrid Policy Optimization** ⭐ [Tencent, 119↑] — arXiv 2512.24615
  - 把 agent 的**自动生成**与**持续优化**端到端整合的开源框架，腾讯优图出品。
  - **问题诊断**：当下 agent 系统的两大痛点是"手工搭建 agent 太贵"与"RL 训练成本极高"，分别需要数百 person-days 和上万美元算力。
  - **方法**——四件套：① **三层 YAML 框架**（Agent/Tools/Env 解耦），用户只需声明式描述，秒级换 LLM 后端；② **自动配置**两路——Workflow（专家模板）+ Meta-Agent（基于用户自然语言需求由 LLM 自动生成 YAML 与 prompt）；③ **Training-free GRPO**——关键算法创新：用 LLM evaluator 把 group rollout 的成败信号蒸馏成"语义 advantage"，注入 textual LoRA（外挂 system prompt + few-shot），实现 $18 击败 $10K 传统 RL 的同等效果；④ **端到端 Agent RL**——基于 Agent-Lightning + VeRL，稳定扩展到 128 GPU 并行 rollout。
  - **结果**：WebShop / GAIA / SWE-bench Lite 等多个基准达到开源 SOTA；Training-free GRPO 在多个 agent 任务相对 baseline 提升 8-15pp。**意义**：把"data synthesis + automated configuration + training-free optimization"做成完整工程闭环，是 2026 H1 工业级 agent 框架的代表作；已成精读 [[05-youtu-agent]]。
- **#7 Recursive Language Models** [MIT, 96↑] — arXiv 2512.24601
  - 提出"递归调用 LM"作为 agent 设计的一等公民范式。
  - **核心观点**：当前 agent 系统普遍把 LLM 作为黑盒嵌入到外部 Python 控制流（while loop + tool call），但其实可以反过来——让 LLM 自身的 token 流递归地调用 LLM 完成子任务，把"思考-决策-执行"统一在生成过程中。
  - **实现**：在 prompt 内提供一个 `recurse(sub_prompt)` 原语，模型可在生成中显式调用自己解决子问题，结果以 token 形式返回；递归调用栈通过 KV-cache 复用实现高效执行。**优势**：① 状态共享更深；② 上下文按需扩展，避免一开始就塞满 prompt；③ 与 multi-agent 系统比，递归 LM 在 single-model 内完成，延迟和成本都低。
  - **实验**：数学多步推理、长文档 QA、计划分解任务上一致优于平铺 CoT 与 ReAct 风格 agent。是与 W18 [[16-recursive-mas]] Recursive Multi-Agent Systems 同源的思想性工作（W18 把这一原则推广到 multi-agent）。
- **#9 Evolving Programmatic Skill Networks** [U. de Montréal, 88↑] — arXiv 2601.03509
  - **演化式 programmatic skill 网络**——把 agent 学到的能力表达为**可被组合的程序化 skill 图**，而非单一 policy 网络。
  - **方法**：① skill 用代码片段 + 自然语言描述定义；② 评估时按图遍历组合调用；③ 演化算法（变异 / 交叉 / 选择）不断扩展 skill 库——成功 trajectory 中频繁出现的 sub-procedure 被抽象为新 skill 节点。**理论支撑**：可组合 skill 网络的表达力 + 学习样本复杂度分析。
  - **实验**：MiniGrid / Crafter / ALFWorld 等长程任务上，相比 monolithic RL policy 在新任务的迁移效率提升 3-5×。**与同期工作的对位**：与 [[11-skillclaw]] SkillClaw（多用户生态级 skill 进化）和 W19 Skill1（单策略协同进化）分别对应不同尺度的 skill 学习范式，本文偏理论/单 agent 起点。

### W03（2026-01-11 至 01-17）

- **#1 Watching, Reasoning, and Searching: A Video Deep Research Benchmark** ⭐ [QuantaAlpha, 214↑] — arXiv 2601.06943
  - **首个 video deep research benchmark**——填补了文本 DR（BrowseComp 等）与多模态 DR（VideoMME 等）之间的空白。
  - **任务设定**：以视频为视觉锚点（如"这段视频是哪起新闻事件"），但**最终答案必须从开放 web 上检索得到**——纯视频信息不够、纯文本信息也不够，必须做"视频→实体/事件→web 搜索→合成答案"的跨模态深度研究。
  - **数据集**：100 个经过严格筛选的样本，全部必须通过两个 ablation——**Web-only 失败**（无视频找不到关键词）+ **Video-only 失败**（无 web 拼不出答案），保证任务真正需要跨模态。**实验对比**：系统比较了 Workflow（固定流程）与 Agentic（自由探索）两种范式 × 多个 MLLM。
  - **反直觉发现**：① **Agentic 不必更好**——强模型如 Gemini-3 在 agentic 模式下能利用自由度有显著增益（+10pp+），但**弱模型如 MiniCPM-V 在 agentic 模式下反而退化 9 分**——多步工具调用放大了底层视觉感知错误；② Workflow 在弱模型上更稳定，Agentic 在强模型上才发挥优势。**意义**：把"video + DR + agentic 三者交叉点"建成可重复评测，并指出 agentic 范式有 floor effect。已成精读 [[04-videodr]]。
- **#4 Thinking with Map: Reinforced Parallel Map-Augmented Agent for Geolocalization** [AGI Lab, 170↑] — arXiv 2601.05432
  - 让 VLM 在**图像地理定位**任务中**像人类一样使用地图**——而非仅靠视觉特征猜。
  - **问题**：现有方法把地理定位当 retrieval 问题（query embedding 找最近邻）；但人类做法是"看图找路标 → 想区域 → 看地图比对 → 缩小范围"，是显式的工具增强推理过程。
  - **方法**：① **Agent-in-the-map loop**——agent 显式调用地图工具（缩放/平移/对照路标），把每一步的地图截图作为新观察；② **两阶段训练**——agentic RL（GRPO 训 multi-step 地图工具使用）+ Parallel TTS（test-time 多个 agent 并行推断后投票）；③ 配套发布 **MAPBench** 基准——覆盖城市、农村、特殊地标场景。
  - **关键数字**：Acc@500m（500 米误差内）从 8.0%（Gemini-3-Pro + Google Map baseline）提到 **22.1%**（提升近 3×）。**意义**：把"thinking with tool"这一抽象原则具象到地图域，证明在专门任务上 agentic 设计可超越 frontier 闭源模型 + 工具的简单组合。
- **#6 Rewarding the Rare: Uniqueness-Aware RL for Creative Problem Solving in LLMs** [MIT, 150↑] — arXiv 2601.08763
  - 解决 RL-trained LLM 的**多样性塌缩**问题——传统 RLHF / RLVR 倾向收敛到单一"最优"答案，扼杀创造性任务所需的解空间多样性。
  - **核心思想**：在 reward 中显式加入**稀有性奖励**——一个候选答案越罕见（在 group 内或历史 buffer 内）得分越高。
  - **方法**：① 用 sentence embedding 计算 rollout 间相似度；② 把"uniqueness score"加权混合到 verifier reward 中；③ 训练目标变为"既要对，也要新"。
  - **实验**：在创意写作、开放问答、数学多解题上，相比 vanilla GRPO 多样性指标（self-BLEU 倒数 / cluster count）提升 30-60%，而准确率不降反升 2-3pp（多解题尤其受益）。
  - **意义**：把 RL 训练从"找最优"转向"找有用的多样集合"，对 deep research、scientific reasoning 等需要多角度推理的任务尤其重要。
- **#7 DeepResearchEval: Automated Framework for Deep Research Task Construction** [Infinity Lab, 127↑] — arXiv 2601.09688
  - DR 任务的**自动构建 + agentic 评估**双重框架——解决"DR benchmark 构造成本高 + 静态评测过时快"的双重痛点。
  - **任务构造**：① persona-driven pipeline——为多种领域专家 persona（律师、医生、记者...）生成真实复杂的研究需求；② **两阶段过滤**——Task Qualification（任务是否真实可解）+ Search Necessity（是否真的需要多源证据，过滤掉单页就能答的）；最终保留只有**多源证据 + 多步推理**才能完成的任务。
  - **评测设计**：① **Adaptive Point-wise Quality Evaluation**——根据任务复杂度动态调整 rubric 权重；② **Active Fact-Checking**——即使 agent 没给出引用，框架也能主动 web 搜索验证关键断言。
  - **结果**：在 GPT-5 / Claude 4.6 / Gemini 3 等 frontier 模型上的得分分布更具区分度，比 BrowseComp 等固定 benchmark 更能反映真实差异；自动构造的任务在多月后仍保持挑战性（不被新模型快速饱和）。
- **#8 Controlled Self-Evolution for Algorithmic Code Optimization** [QuantaAlpha, 115↑] — arXiv 2601.07348
  - 解决**自演化代码优化（如 AlphaEvolve）的"探索效率低"**问题——纯随机变异+保留最优的方案在复杂程序空间中样本效率太差。
  - **CSE 三组件**：① **Diversified Planning Initialization**——多个不同"规划方向"的起点同时启动，避免陷入单一局部最优；② **Genetic Evolution**——用反馈信号引导变异 / 交叉（如 profiler 输出指导哪部分变），取代盲目随机；③ **Hierarchical Evolution Memory**——任务内记忆（哪些 mutation 在当前任务有效）+ 任务间记忆（跨任务知识迁移）。
  - **实验**：EffiBench-X（算法效率优化基准）上 vs AlphaEvolve / Eureka / FunSearch 等所有强基线全面提升 15-30%，达到 SOTA；样本效率（达到目标 quality 所需 evaluation 次数）提升 5-10×。
  - **意义**：把"自演化代码"从蒙特卡洛搜索升级为带先验的引导搜索，是该方向工程效率的关键一步。
- **#10 MAXS: Meta-Adaptive Exploration with LLM Agents** [Xi'an Jiaotong U., 96↑] — arXiv 2601.09259
  - 解决 LLM agent 探索时的两个根本问题：**局部短视生成**（只看下一步不看多步收益）与**轨迹不稳定性**（同一情境多次 rollout 行为差异大）。
  - **MAXS 方法**：① **Lookahead 优势估计**——前瞻 K 步用 LLM 模拟"如果用工具 A vs 工具 B 接下来会如何"，估计每个工具的预期增益；② **稳定性筛选**——计算 step consistency variance（多次 rollout 在该步的行为分散度）+ trend slopes（reward 累积趋势）联合识别稳定步骤；③ **轨迹收敛机制**——把多条 rollout 在稳定步骤前合并，减少无效分支。
  - **实验**：3 个 base LLM × 5 个 dataset（数学/编程/工具使用/web 搜索/具身任务）一致优于 ReAct / Reflexion / Tree-of-Thought 等基线；尤其在长时程任务（>20 步）效果更显著。
  - **意义**：把"探索"从启发式调参转为可量化的"前瞻+稳定性"双指标驱动。

### W04（2026-01-18 至 01-24）

- **#1 Agentic Reasoning for Large Language Models** [UIUC, 205↑] — arXiv 2601.12538
  - **agentic reasoning 大型综述**——2026 H1 该领域最权威的体系化梳理。
  - **三层架构论**：① **Foundational layer**（单 agent 核心能力）——规划（planning）、工具使用（tool use）、搜索（search）、记忆（memory）；② **Self-evolving layer**——基于反馈/记忆/适应能力的自我改进，含 reflection、self-critique、experience accumulation；③ **Collective layer**——multi-agent 协作（debate / role-play / market / hierarchy 等组织形式）。
  - **第二维度分类**：区分 **in-context reasoning**（结构化 prompt 编排，无需训练）与 **post-training reasoning**（SFT + RL 内化能力，需要训练）——两条路线对应不同成本-效果曲线。
  - **覆盖范围**：400+ 论文综合，从 ReAct、ToT 到 OpenAI o1、Gemini Deep Think、Claude Code 全部纳入；附 evaluation taxonomy 与开放挑战。
  - **意义**：奠定 agentic reasoning 的术语体系与分类框架；被后续 2026 多篇综述 / position paper 引为基础。
- **#2 Your Group-Relative Advantage Is Biased** [13 作者, 158↑] — arXiv 2601.08521
  - 揭示 **GRPO 算法的根本系统性偏差**——一个被广泛使用但少有人怀疑的算法基础有显著问题。
  - **核心发现**：GRPO 把 group 内 reward 标准化为 advantage，但这导致**对难题系统性低估优势（因 group 内大多失败，正确答案的 advantage 被分母 std 拉小）、对简单题系统性高估优势（因 group 内大多成功，advantage 集中在少数 fail 上）**——梯度信号严重失衡。
  - **方法**——**History-Aware Adaptive Difficulty Weighting (HA-DW)**：① 维护演化的题目难度锚点（用历史成功率估计）；② 基于训练动态自适应重加权 advantage——难题增益、简单题抑制。
  - **实验**：在 5 个数学基准（GSM8K、MATH、AIME、AMC、Omni-MATH）全面提升 1-4 pp；Qwen2.5-Math-7B 训练曲线更稳。
  - **意义**：是 GRPO 系列算法（[[01-grandcode]] Agentic GRPO、[[08-sdar]] SDAR、[[09-delta]] DelTA）的"地基修正"工作，所有后续 GRPO 变体应叠加该修正。
- **#3 EvoCUA: Evolving Computer Use Agents** [Meituan, 92↑] — arXiv 2601.15876
  - 美团出品的**自维持进化式 computer use agent** 训练系统。
  - **完整循环**：① **可验证合成引擎**——大批量生成可自动判分的 CUA 任务（GUI 操作、网页表单、文件操作等）；② **数万异步沙箱并行 rollout**——大规模 docker 容器集群让 agent 同时尝试上万任务，效率比同步训练高 100×；③ **迭代进化机制**——成功 trajectory 加入 positive buffer 强化训练；失败 trajectory 转为监督信号（让 reviewer agent 标注失败原因，作为新数据）。
  - **关键数字**：**OSWorld 56.7%** 达到开源 SOTA，超越 UI-TARS-2 (53.1%) 和 OpenCUA-72B (45.0%)。
  - **意义**：把"环境自动合成 + 大规模并行 RL + 失败数据回收"整合为可持续运转的工业级 CUA 生产线，是 [[02-termigen]] TermiGen 思路在 GUI 域的极致扩展。
- **#4 LLM-in-Sandbox Elicits General Agentic Intelligence** [Microsoft Research, 87↑] — arXiv 2601.16206
  - **极简但深刻的论点**：把计算机虚拟化为**仅含基础功能的代码沙箱**（无浏览器、无 GUI、无网络，只有 Python REPL + 文件系统 + shell），就能激发 LLM 通用任务求解元能力。
  - **核心论点**：复杂 agent 框架（LangChain、AutoGen 等）大量的脚手架其实把 LLM 自身的元能力封装隐藏了；把这些复杂 API 还原为"代码 + 文件 + 执行"三件套，LLM 反而能自由发挥。
  - **三大元能力**：① 外部资源访问（curl/wget/git）；② 文件管理；③ 代码执行。
  - **LLM-in-Sandbox-RL 训练方法**：用**非 agentic 数据**（普通问答 + 代码任务）让模型学会"在 sandbox 内自由折腾"，无需特定 agent 数据。
  - **结果**：强模型（如 GPT-5）+15.5% 任务通过率，**token 消耗 -8×**——精简反而更高效。
  - **意义**：质疑了"agent 必须复杂 harness"的主流假设，给出"sandbox + RL"的极简替代。
- **#9 ABC-Bench: Benchmarking Agentic Backend Coding** [OpenMOSS, 67↑] — arXiv 2601.11077
  - **后端开发的 agent 评测**——区别于 SWE-Bench 主要考察"读代码改 bug"，ABC-Bench 评估"从零搭建并部署一个后端服务"。
  - **数据规模**：**224 个真实任务 × 8 编程语言（Python/Java/Go/Rust/JS/TS/C#/PHP）× 19 后端框架**（Django/Spring/Gin/Actix/Express/...）。
  - **任务设计**：每个任务要求 agent 完整执行"仓库探索 → 编写代码 → 容器化（写 Dockerfile）→ 启动服务 → 通过外部 API 集成测试"全生命周期。
  - **关键发现**：即使最强模型（GPT-5 + Claude Code）也难以可靠完成大多数任务，平均通过率不到 25%；**容器化**是失败高发环节（依赖版本、端口冲突、环境变量），暴露 agent 对系统级运维的不熟悉。
  - **意义**：把 coding agent 评估从"修代码"推到"运维 + 部署"，揭示 SWE 系列基准未覆盖的实战盲区。

  - **任务设计**：选取 10 个学科领域 × 40 篇真实顶会论文，让 agent 从 abstract+motivation 出发，**完整复现整个研究**——文献调研、假设提出、实验设计、代码实现、数据分析、论文撰写。
  - **评测维度**：研究新颖性、实验严谨性、结果可靠性、写作质量、与原论文的对齐度。**配套环境**：Docker 沙盒 + 可执行 dataset 接口 + 自动 reviewer pipeline（多 LLM 评审 + reference paper 对比）。
  - **关键发现**：① 即使顶级模型 + 强 harness（AutoResearchClaw）端到端通过率也极低（约 5-15%）；② agent 在"做实验"这步失败率最高（**>60%** 因环境 / 依赖 / 复现细节出错）；③ AI 写作语言流畅但"科研逻辑"常断裂。揭示了 agent science 与真实科研的 gap 远比 ARC-Bench 显示的更大。
- **#8 Role-Agent: Bootstrapping LLM Agents via Dual-Role Evolution** [7 作者 (USTC + 高德), 73↑] — arXiv 2606.04088
  - 解决 agent RL 训练**真实环境难获取**的根本痛点。
  - **核心创新**——**让单一 LLM 同时扮演 agent 和 environment 两个角色**：role A 作为 task solver（出 action），role B 作为 environment simulator（出 observation / reward / next state），两者交替演化。**自举循环**：每一轮 agent 学会更优策略 → environment 也学会更具挑战性的响应 → 反过来再训 agent。**约束**：environment role 用 reward model 保证物理合理性 + retrieval grounding 防止幻觉。
  - **结果**：基于 Qwen3-8B 在 WebShop / ALFWorld / Travel-Planner 三基准平均 **+4%**，且**无需任何真实环境部署**；论文证明 dual-role 演化在分布上与真实环境训练同效。是"环境合成 → 直接让模型自己当环境"路线的极简实现。
- **#9 On the Geometry of On-Policy Distillation** [HKUST 5 作者, 62↑] — arXiv 2606.03950
  - **OPD (On-Policy Distillation)** 的训练动力学分析。
  - **核心发现**——OPD 存在 **"subspace locking"** 现象：student 模型快速被锁进 teacher 的某个低维 logit 子空间，丧失对其他方向的探索能力，导致 long-horizon agent 任务上 OPD 反而**不如普通 SFT**。**几何分析**：把 KL divergence 分解到 logit 主成分上，证明 OPD 的梯度在前 k 主方向集中，其余方向几乎为零。**改进方案**：geometry-aware OPD——在前 k 方向加正则、释放后 N-k 方向自由度。
  - **实验**：在 reasoning RL + agent RL 多任务上修正后 OPD 一致超越基线 2-5pp。理论价值高于工程价值，但为后续 OPD-based agent 训练提供了诊断工具。
- **#10 Latent Spatial Memory (Mirage) for Video World Models** [Microsoft Research, 58↑] — arXiv 2606.04129
  - 视频 world model 的**潜空间 3D 记忆**机制——专为 long-horizon 生成 / 交互式 world model 设计。**问题**：现有 video world model（如 Genie-3、Sora）必须保留完整历史帧才能保持空间一致性，长视频内存爆炸。**Mirage 方案**：把场景压缩到**潜空间 3D voxel grid** 持续维护——每一帧生成时更新对应 voxel，新视角生成时从 voxel grid 解码，无需保留原始帧。
  - **关键数字**：① 速度 **10.57×**（无须重算历史 attention）；② 显存 **-55×**（voxel grid 比帧序列小 50 倍以上）；③ 一致性指标比 Genie-3 baseline **+12%**。**意义**：把"世界状态"作为一阶 object 显式建模，是 world model 从"frame predictor"走向"真正的空间记忆"的重要一步；与 [[14-gamma-world]] 的多 agent 路线互补。
