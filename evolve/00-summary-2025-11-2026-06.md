## 一、按月列出（所有摘要基于 arXiv 原文）

### 2025-11（6 篇）

- **#14 GigaEvo: An Open Source Optimization Framework Powered By LLMs And Evolution Algorithms** 📄 [AIRI, 121↑] — arXiv 2511.17592 → 见 [[15-gigaevo]]
  - **定位**：AlphaEvolve 的**开源可复现实现**——原论文实现细节不全，GigaEvo 把关键组件全部模块化开源。
  - **组件**：MAP-Elites 质量-多样性算法、异步 DAG 评估流水线、LLM 驱动的变异算子（带 insight 生成 + 双向 lineage 追踪）、灵活的 multi-island 演化策略。
  - **验证**：在 AlphaEvolve 论文的挑战问题（Heilbronn 三角放置、圆填充、高维 kissing number）上复现，验证实现正确性。**意义**：为"LLM × 进化算法"社区提供可复现基建。

- **#17 Agent0: Unleashing Self-Evolving Agents from Zero Data via Tool-Integrated Reasoning** 📖 [UNC, 110↑] — arXiv 2511.16043 → 精读 [[01-agent0]]
  - **零数据自演化奠基作**——curriculum agent 出题、executor agent 解题，从同一 base LLM 初始化，**多步共生竞争**。
  - **工具集成是关键**——工具打破 base model 能力天花板，让 co-evolution 不早熟。
  - **结果**：Qwen3-8B-Base 数学 +18%、通用推理 +24%，全程零外部数据。1.2k★。

- **#29 AutoEnv: Automated Environments for Measuring Cross-Environment Agent Learning** 📄 [FoundationAgents, 92↑] — arXiv 2511.19304 → 见 [[16-autoenv]]
  - **指出盲区**：现有 agent 多在**单一域内**自演化，隐含"环境分布固定"假设；**跨环境学习**几乎无人测。
  - **方法**：① AutoEnv——把环境当作"转移/观察/奖励的可因子化分布"，平均 **$4.12** 低成本生成异构世界；② 构建 AutoEnv-36（36 环境、358 验证关卡）；③ 把 agent 学习形式化为 Selection-Optimization-Evaluation 三阶段的组件化过程。
  - **结果**：7 个 LM 只拿 12-49% 归一化奖励——跨环境学习是真实挑战。

- **#33 Scaling Agent Learning via Experience Synthesis（DreamGym）** 📖 [Meta, 83↑] — arXiv 2511.03773 → 精读 [[04-dreamgym]]
  - **首个统一经验合成框架**——把环境动态蒸馏为"基于推理的经验模型"，在合成经验里训 agent。
  - **三件套**：reasoning-based experience model + experience replay buffer（真实数据初始化）+ 自适应任务生成（在线课程）。
  - **结果**：WebArena 超基线 30%+；纯合成追平 GRPO/PPO；sim-to-real warm-start 大幅减少真实交互。

- **#69 Agent0-VL: Exploring Self-Evolving Agent for Tool-Integrated Vision-Language Reasoning** 📄 [UNC, 49↑] — arXiv 2511.19900 〔分类⑦〕
  - **[[01-agent0]] 的视觉版**——把自演化推进到 VLM。针对"纯文本 self-evaluation 无法验证复杂视觉推理步骤、易产生评估幻觉"的痛点。
  - **方法**：把**工具集成**不只用于推理，还用于**self-evaluation 与自我修正**——agent 用工具核验自己的视觉推理，实现持续改进。
  - **意义**：self-evolution × tool-integrated × multimodal 三者交叉，缓解视觉自评幻觉。

- **#75 VisPlay: Self-Evolving Vision-Language Models from Images** 📄 [UIUC, 45↑] — arXiv 2511.15661 〔分类⑦〕
  - **从无标注图像自演化的 VLM**——无需人工标注或任务专属 heuristics 定义可验证奖励。
  - **双角色 self-play**：Image-Conditioned Questioner（出可答的难题）+ Multimodal Reasoner（产 silver 答案），用 GRPO 联合训练。
  - **意义**：把 [[01-agent0]] 式双角色自博弈落地到视觉模态，规模化利用海量无标注图像。57★。

### 2025-12（2 篇）

> 12 月 top-50 中无明确以"agent 自演化"为核心的论文，但 **top-100 中出现 2 篇**（其余命中多为模型技术报告、扩散语言模型、deep research，已归入其它目录）。

- **#58 Guided Self-Evolving LLMs with Minimal Human Supervision** 📄 [腾讯, 55↑] — arXiv 2512.02472 〔分类①〕
  - **直面自演化的"塌缩"问题**——无引导的自演化系统常**快速 plateau 甚至退化**，根因是 concept drift、diversity collapse、mis-evolution（模型强化自身偏见、收敛到低熵行为）。
  - **方法 R-Few**：guided Self-Play Challenger-Solver 框架，通过 in-context grounding + mixed training 引入**轻量人类监督**，让自演化稳定可控。
  - **意义**：自演化稳定性问题的代表性诊断 + 解法，与 [[08-ctx2skill]] 的防对抗崩溃同主题。

- **#96 Reinforcement Learning for Self-Improving Agent with Skill Library（SAGE）** 📄 [42↑] — arXiv 2512.17102 〔分类②〕
  - **用 RL（而非纯 prompting）系统性把 skill 纳入学习**——现有 skill library 方法多靠 LLM prompting，实现不稳定。
  - **方法 SAGE**（Skill Augmented GRPO for self-Evolution）：把 skill 系统性融入 GRPO 学习框架，让 agent 边学边用边验证新 skill。
  - **意义**：是 [[05-skillrl]]/[[06-skill1]] 同主题的早期工作（skill 库与 RL 结合），强调"一致的 skill 实现"。

### 2026-01（8 篇）

- **#4 Agentic Reasoning for Large Language Models** 📖 [UIUC, 206↑] — arXiv 2601.12538 → 精读 [[21-agentic-reasoning-survey]]
  - **本库最高票综述**——提出 agentic reasoning 三层框架：**foundational → self-evolving → collective**。
  - **Self-Evolving Layer** 被正式确立为独立研究层（通过 feedback/memory/adaptation 精炼能力）——本库几乎所有论文都落在这一层。
  - 第二维度：**in-context（不训练）vs post-training（训练）**。配套 1.3k★ Awesome 列表。

- **#23 Controlled Self-Evolution for Algorithmic Code Optimization（CSE）** 📖 [QuantaAlpha, 115↑] — arXiv 2601.07348 → 精读 [[13-cse]]
  - 诊断自演化代码优化的**探索效率低**——三根因：初始化偏置、随机操作无反馈、经验利用不足。
  - **三招**：多样化规划初始化 + 反馈引导的遗传进化（定向变异 + 组合交叉）+ 层级化进化记忆（任务内 + 任务间）。
  - **结果**：EffiBench-X 跨多 LLM backbone 全面超基线，早期就高效且持续改进。

- **#35 EvoCUA: Evolving Computer Use Agents via Learning from Scalable Synthetic Experience** 📖 [美团, 92↑] — arXiv 2601.15876 → 精读 [[02-evocua]]
  - 把 CUA 的**数据生成 ⇄ 策略优化**合并为自持演化循环——可验证合成引擎 + 数万异步 sandbox rollout + 按能力边界调控的迭代演化学习。
  - **失败轨迹 → 富监督信号**（error analysis + self-correction）。
  - **结果**：OSWorld **56.7%** 刷新开源 SOTA，反超闭源 UI-TARS-2（53.1%）。

- **#37 Evolving Programmatic Skill Networks（PSN）** 📖 [蒙特利尔大学, 88↑] — arXiv 2601.03509 → 精读 [[07-psn]]
  - **程序化 skill 进化派**——skill = 可执行符号程序，组成可组合网络随经验进化。
  - **三机制**：REFLECT 结构化故障定位 + maturity-aware update gating（稳定性 vs 可塑性平衡）+ 规范化重构 + 回滚验证。
  - **洞察**：PSN 学习动态与神经网络训练**结构同构**。MineDojo/Crafter 强复用、快适应、强泛化。

- **#73 ArenaRL: Scaling RL for Open-Ended Agents via Tournament-based Relative Ranking** 📄 [阿里 NLP, 54↑] — arXiv 2601.06487 〔分类④〕
  - **针对开放式 agent 任务**（如复杂旅行规划，解空间巨大、无客观 ground-truth）——指出 pointwise scoring 的 **discrimination collapse**（reward 被压缩到窄区间，信号被噪声淹没）。
  - **方法**：用**锦标赛式相对排名**（tournament-based relative ranking）替代标量打分，放大轨迹间的细微优势。
  - **意义**：为"开放式、无标准答案"任务的演化式 RL 提供可扩展的相对评价信号。254★。

- **#81 UniCorn: Towards Self-Improving Unified Multimodal Models through Self-Generated Supervision** 📄 [中科大, 51↑] — arXiv 2601.03193 〔分类⑦〕
  - **统一多模态模型的自改进**——提出 "Conduction Aphasia"（能理解多模态输入却难转化为可控生成）。
  - **方法**：把单个 UMM 切成 **Proposer-Solver-Judge 三角色**，self-play 产高质量交互 + cognitive pattern reconstruction 蒸馏潜在理解，**无需外部数据/teacher**。
  - **意义**：self-play 三角色范式在"理解→生成"鸿沟上的应用。173★。

- **#88 Beyond Static Tools: Test-Time Tool Evolution for Scientific Reasoning** 📄 [上海 AI Lab, 48↑] — arXiv 2601.07641 〔分类②〕
  - **工具也要进化**——科学领域工具稀疏、异构、本质不完备，静态工具库范式失效。
  - **方法 TTE**（Test-Time Tool Evolution）：让 agent 在**推理时合成、验证、进化可执行工具**，把工具从固定资源变为问题驱动的产物。配套 SciEvo 基准（1,590 科学推理任务）。
  - **意义**：把"进化"从 skill 扩展到**工具**——test-time 动态造工具。

- **#95 Learning to Discover at Test Time（TTT-Discover）** 📄 [Stanford, 45↑] — arXiv 2601.16175 〔分类③〕
  - **在测试时做 RL 来发现 SOTA**——区别于 AlphaEvolve（prompt 冻结的 LLM 搜索），TTT-Discover 让 LLM 在测试问题上**继续训练**，积累该问题专属经验。
  - **特殊的持续学习**：目标是"产出一个极好的解"而非"平均很多好解"，为此设计了优先最有希望解的学习目标与搜索子程序。
  - **意义**：把 test-time training 与演化式发现结合，是"为单一难题专门进化"的代表。585★。

### 2026-02（7 篇）

- **#17 QuantaAlpha: An Evolutionary Framework for LLM-Driven Alpha Mining** 📖 [QuantaAlpha, 190↑] — arXiv 2602.07085 → 精读 [[14-quantaalpha]]
  - 演化式 agent 落地**量化金融**——每次挖掘当作 trajectory，做**轨迹级变异/交叉**，复用高奖励片段。
  - **语义一致性约束**（假设↔表达式↔代码）+ 反 crowding 复杂度约束。
  - **结果**：CSI 300 上 IC 0.1501、ARR 27.75%、MDD 7.98%；跨市场迁移 CSI 500/S&P 500 四年超额 160%/137%。1.1k★。

- **#42 SkillRL: Evolving Agents via Recursive Skill-Augmented Reinforcement Learning** 📖 [UNC, 76↑] — arXiv 2602.08234 → 精读 [[05-skillrl]]
  - 经验蒸馏成层级 SkillBank（存模式不存轨迹）+ 自适应检索 + **skill 库与 policy 递归共进化**。
  - 与 [[01-agent0]] 同组（aiming-lab），把自演化从推理层推进到 skill 层。
  - **结果**：ALFWorld/WebShop/7 个 search 任务超基线 +15.3%，token 占用大幅下降。833★。

- **#52 MARS: Modular Agent with Reflective Search for Automated AI Research** 📄 [Google, 67↑] — arXiv 2602.02660 〔分类⑥〕
  - **自动化 AI 研究的反思式 agent**——AI 研究评估昂贵（要训模型）、归因不透明。
  - **三支柱**：Budget-Aware Planning（成本约束的 MCTS）+ Modular Construction（Design-Decompose-Implement 管线）+ **Comparative Reflective Memory**（对比分析解的差异做 credit assignment）。
  - **意义**：把"反思 + 记忆"用于 agent 设计研究流程，自改进研究能力。

- **#55 MemSkill: Learning and Evolving Memory Skills for Self-Evolving Agents** 📄 [南洋理工, 63↑] — arXiv 2602.02474 〔分类⑤〕
  - **把记忆操作变成可学习、可进化的"记忆 skill"**——现有 memory 系统靠少量静态手工操作，硬编码了人类先验，僵化且低效。
  - **方法**：受 agent skill 哲学启发，controller 学习选择少量相关"记忆 skill"（提取/巩固/剪枝信息的结构化可复用例程），LLM executor 产出 skill-guided 记忆。
  - **意义**：把"skill 进化"思想注入**记忆系统**——记忆操作本身也能进化。506★。

- **#61 SWE-Universe: Scale Real-World Verifiable Environments to Millions** 📄 [Qwen, 61↑] — arXiv 2602.02361 〔分类④〕
  - **从 GitHub PR 自动构建百万级可验证 SWE 环境**——克服自动构建的低产出、弱验证、高成本三难。
  - **方法**：building agent（自训高效模型）用 **iterative self-verification + in-loop hacking detection** 保证生成高保真可验证任务，规模化到 **807,693** 个多语言环境。
  - **意义**：为"环境协同进化"提供百万级真实环境基建，支撑 agentic mid-training + RL。

- **#67 SkillOrchestra: Learning to Route Agents via Skill Transfer** 📄 [威斯康星大学, 58↑] — arXiv 2602.19672 〔分类②〕
  - **skill-aware 的 compound AI 编排**——既有 router 做粗粒度 query 级决策、忽略演化中的任务需求；RL 编排器易"routing collapse"。
  - **方法**：不直接端到端学路由策略，而是从执行经验**学习细粒度 skill**，建模各 agent 在不同 skill 下的能力与成本，部署时按 skill 需求路由。
  - **意义**：把 skill 学习用于**多 agent 编排**，缓解路由塌缩。64★。

- **#82 daVinci-Agency: Unlocking Long-Horizon Agency Data-Efficiently** 📄 [SII-GAIR, 53↑] — arXiv 2602.02619 〔分类④〕
  - **长程 agency 的数据合成**——核心瓶颈是缺乏捕捉"长依赖结构 + 跨阶段演化动态"的训练数据。
  - **关键洞察**：把数据合成重新理解为**真实软件演化**——PR（Pull Request）序列天然蕴含长程学习的监督信号（把复杂目标分解为可验证提交单元）。
  - **意义**：用"软件演化轨迹"作为长程 agent 的高质量监督来源。

### 2026-03（5 篇）

- **#41 SkillNet: Create, Evaluate, and Connect AI Skills** 📄 [浙大, 94↑] — arXiv 2603.04448 → 见 [[10-skillnet]]
  - **20 万 skill 的开源基础设施**——解决 agent "reinvent the wheel"问题。
  - 统一 ontology 组织 skill，支持从异构来源创建 skill、建立关系连接、做 **5 维评估**（Safety/Completeness/Executability/Maintainability/Cost）。
  - **结果**：ALFWorld/WebShop/ScienceWorld 上平均奖励 +40%、执行步数 -30%。集成 20 万+ skill 仓库 + 交互平台 + Python 工具包。1.0k★。

- **#68 Online Experiential Learning for Language Models（OEL）** 📄 [Microsoft Research, 60↑] — arXiv 2603.16856 〔分类①〕
  - **从部署经验在线持续改进**——现有范式靠离线训练，部署期积累的丰富经验被完全浪费。
  - **方法 OEL**：两阶段——① 从用户侧交互轨迹提取并积累**可迁移经验知识**；② 通过 **on-policy context distillation 把经验巩固进模型参数**（无需访问用户侧环境），两阶段迭代成在线学习闭环。
  - **意义**：把"部署即学习"形式化——经验进权重，呼应 [[18-evoarena]] 的记忆视角与本库的经验主线。

- **#74 Memento-Skills: Let Agents Design Agents** 📄 [UCL, 58↑] — arXiv 2603.18743 〔分类⑥〕
  - **agent 设计 agent**——通用、可持续学习的系统，自主构造/适配/改进任务专属 agent。
  - **方法**：基于 memory-based RL + stateful prompts，可复用 skill 以**结构化 markdown 文件**作为持久、进化的记忆；从简单 skill（web 搜索/终端）出发，通过 **Read-Write Reflective Learning** 持续改进。
  - **意义**：把进化单位提升到"agent 设计 agent"，skill = 进化记忆。1.5k★。

- **#83 MM-Zero: Self-Evolving Multi-Model Vision Language Models From Zero Data** 📄 [NVIDIA, 54↑] — arXiv 2603.09206 〔分类⑦〕
  - **首个零数据自演化的 VLM 推理框架**——VLM 因视觉模态通常需种子数据，MM-Zero 突破这一限制。
  - **方法**：超越 dual-role（Proposer/Solver），引入**三专门角色**的 multi-role 自演化训练框架，从零 data 自演化 VLM 推理。
  - **意义**：把 [[01-agent0]] 式零数据自演化彻底推广到多模态。75★。

- **#88 Hyperagents** 📄 [Meta FAIR, 51↑] — arXiv 2603.19461 〔分类⑥〕
  - **自指（self-referential）元进化**——延续 Darwin Gödel Machine（DGM）的开放式自我改进思想。
  - **诊断**：DGM 在 coding 域有效是因为"评估和自我修改都是 coding 任务"，能力增益可转化为自我改进增益；但**这一对齐在 coding 域外不成立**。
  - **方法**：hyperagents = task agent（解目标任务）+ meta agent（改进 task agent）的自指整合，把自我改进推广到非 coding 域。**意义**：元自改进（meta-self-improvement）的前沿探索。2.6k★。

  - **意义**：把自演化推到"无监督、无 verifier"的最难场景——open-world bootstrap。48★。

### 1️⃣ "Zero-Data / 自造数据 / Open-World Bootstrap"成为主旋律

- 共识：**环境是驱动能力持续进化的关键**——不能只优化模型

### 3.1 发展现状：八个月走过的三个阶段（top-100 视角，50 篇）

把 2025-11 ~ 2026-06 的 **50 篇**按时间铺开，"agent evolve"主题清晰地走过了**三个阶段**：

- **阶段①（2025-11~12）单点突破 + 初步扩散**：[[01-agent0]]、[[04-dreamgym]] 证明"零数据/合成经验自演化"可行；同期 Agent0-VL/VisPlay 已把范式扩到 VLM；12 月 R-Few 率先诊断"自演化塌缩"。
- **阶段②（2026-01~02）范式确立 + 多模态扩散**：[[21-agentic-reasoning-survey]] 综述确立 self-evolving layer；三大流派代表作密集出现；MM-Zero/UniCorn 把零数据自演化推广到多模态；SWE-Universe 把环境推到百万级。
- **阶段③（2026-03~06）体系化 + 治理 + 元进化 + 最新前沿**：关键词从"能不能进化"转向"如何**稳定、可治理、可递归**地进化"——[[09-skillsvote]] 治理、[[08-ctx2skill]] 防崩溃、Hyperagents/Memento 元进化、Need Sleep 自我修改、OpenSkill 开放世界 bootstrap、[[22-agentic-env-survey]] 综述收束。

**当前成熟度判断**：
- **已解决（从研究角度）**：① 零数据/合成数据自演化的**可行性**（含多模态）；② skill 作为能力载体的**三种实现路径**；③ co-evolution 的**基本稳定性**（防崩溃/防污染门控）；④ 自演化向 VLM/空间/GUI 的**可移植性**。
- **正在解决**：① 自演化的**可治理性**（[[09-skillsvote]]、R-Few）；② 动态环境下的**记忆/评测**（[[18-evoarena]]、MemSkill）；③ 进化单位向**组织级 / agent 设计 agent / 元进化**扩展（[[19-onemancompany]]、Memento、Hyperagents）；④ **开放世界无 verifier** 的 bootstrap（OpenSkill、SCOPE）。
- **尚未解决**：见下 3.3。

### 3.3 未来方向趋势（基于 50 篇的缺口与多篇综述的展望）

> 综合本库论文的局限（见各篇第 4 节）+ [[21-agentic-reasoning-survey]] / [[22-agentic-env-survey]] 等综述明确提出的 open problems。

**方向 1️⃣：从"单任务/单域演化"走向"终身、跨域演化（Lifelong & Cross-domain）"**
- 现状：多数工作仍在单域（ALFWorld/WebShop/OSWorld）内演化；[[16-autoenv]] 已揭示**跨环境学习几乎未被解决**（7 个 LM 只拿 12-49%）。
- 趋势：真正的 lifelong learning——agent 在持续变化的多域环境里不遗忘、可迁移。[[18-evoarena]] 的"演化链评测"是第一步。

**方向 2️⃣：自演化的"可治理性 / 安全性 / 对齐"成为必答题**
- 现状：[[09-skillsvote]] 已把"治理"独立出来；[[21-agentic-reasoning-survey]] 明确把 **governance** 列为未来方向。
- 趋势：当 agent 能自主进化能力，**如何防止能力漂移、目标偏移、skill 库被恶意/低质内容污染**会成为核心议题——evidence-gated / validation-gate 只是开始。

**方向 3️⃣：进化"单位"继续上移——从组织到多组织生态**
- 现状：进化单位已从 能力→skill→环境→记忆→**组织**（[[19-onemancompany]]）。
- 趋势：[[22-agentic-env-survey]] 提出 **Multi-agent Environments**；下一步是**多 agent 组织/市场级别的协同进化**——agent 之间形成分工、交易、竞争的演化生态。

**方向 4️⃣：Environment-as-a-Service（环境即服务）+ Neural-Symbolic 环境**
- 现状：[[03-agent-world]] 用 MCP 接入真实环境已是雏形；[[22-agentic-env-survey]] 明确提出 EaaS 与 Neural-Symbolic 环境。
- 趋势：环境像云服务一样**可按需合成、计费、复用**；符号（可验证）与神经（可生成）环境融合，兼顾保真度与多样性。

**方向 5️⃣：自演化的"理论化"——可证明的收敛性与稳定性**
- 现状：[[07-psn]] 发现"skill 进化 ≈ 神经网络训练"的同构，HF SkillOpt 把 SGD 搬到文本空间——都还是**经验类比**，缺乏理论保证。
- 趋势：把文本/符号空间的演化与优化理论对接，给出**收敛性、稳定性、样本复杂度**的可证明结果——这是该领域从"工程"走向"科学"的关键。

**方向 6️⃣：World Model 与自演化深度融合**
- 现状：[[04-dreamgym]] 的"reasoning-based experience model"本质是轻量 world model；[[21-agentic-reasoning-survey]] 把 **world modeling** 列为深水区。
- 趋势：用更强的 world model 生成更真实的"想象经验"，让 agent 在"脑内沙盘"里大规模演化——sim-to-real 的保真度是关键瓶颈。

**方向 7️⃣：评测体系跟上——动态、演化、经济价值导向**
- 现状：[[18-evoarena]] 揭示现有 agent 在动态环境仅 39.6%；静态基准（GAIA/HLE）已饱和。
- 趋势：评测从"静态难题"转向"**持续演化 + 真实经济价值**"（呼应 huggingface 库的 Agents' Last Exam）——能否在不断变化的真实环境里**持续保持**能力，成为新标尺。

**方向 8️⃣：效率与成本——让自演化"用得起"**
- 现状：[[02-evocua]] 数万 sandbox rollout、[[14-quantaalpha]] 依赖 GPT-5.2、co-evolution 算力高——**门槛是普及的最大障碍**。
- 趋势：[[04-dreamgym]] 的"合成 warm-start + 少量真实 RL"、frozen model + 外部状态进化（[[09-skillsvote]]、SkillAdaptor）等**低成本路径**会持续受重视；training-free 演化是重要补充。

**方向 9️⃣：递归 / 元自改进（Recursive & Meta Self-Improvement）——从"改能力"到"改改进机制"**
- 现状：Hyperagents（Darwin Gödel Machine 思路）已指出"自我改进能力在 coding 域外难对齐"；Memento-Skills 让 agent 设计 agent；Need Sleep 让模型自我修改、巩固。
- 趋势：真正的递归自我改进——agent 不只解题、还改进自己的学习算法/元机制。这是通往"超级智能"叙事的核心，也是最难、最具风险的方向（[[21-agentic-reasoning-survey]] 列为深水区）。

**方向 🔟：开放世界 Bootstrap + 自演化的稳定性理论统一**
- 现状：OpenSkill/SCOPE 把自演化推到"无 verifier、无成功轨迹、只给 prompt"的开放世界；但 R-Few、[[08-ctx2skill]]、Guided Self-Evolving 都揭示**无引导自演化会塌缩**（concept drift / diversity collapse / mis-evolution）。
- 趋势：把"开放世界自造学习循环"与"防塌缩稳定性"统一——需要既能自举又不退化的机制。这是"自造数据"主线走到尽头后的必答题。

### 3.4 一句话总结现状与方向

> **现状**：2026 H1 的 agent evolve 已完成"从能不能进化 → 各流派范式确立 + 向多模态/空间/GUI 全面扩散 → 稳定可治理地进化 + 元进化萌芽"的三级跳；**skill 进化 + co-evolution + 防崩溃门控**是已成型的主流配方，**zero-data / open-world bootstrap**是最激进的前沿。
>
> **方向**：下一阶段的胜负手在 **① 终身跨域、② 可治理/可对齐、③ 进化单位上移到生态/元机制、④ 环境即服务、⑤ 理论化、⑥ world model 融合、⑦ 动态评测、⑧ 低成本、⑨ 递归/元自改进、⑩ 开放世界 bootstrap + 防塌缩**——谁能把"自演化"做得**又稳、又省、又可控、还能自我递归**，谁就能定义 2026 H2 乃至通向 superintelligence 的范式。
