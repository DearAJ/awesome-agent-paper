#### #14 GigaEvo: An Open Source Optimization Framework Powered By LLMs And Evolution Algorithms 📄 [AIRI, 121↑] — arXiv 2511.17592 → 见 [[15-gigaevo]]〔分类③〕

**问题动机**：Google 的 AlphaEvolve 展示了"LLM + 进化算法"在数学/算法优化上的强大威力，但其**已发表的描述是高层次的、留下大量实现细节未明**，阻碍了社区的可复现性与后续研究。开源社区缺一个把 AlphaEvolve 关键组件完整、模块化实现的基建。

**核心方法**：GigaEvo（AIRI 出品）是一个**开源的 LLM-进化混合优化系统**，把 AlphaEvolve 的关键组件全部模块化开源：① **MAP-Elites 质量-多样性算法**——维护一个按行为特征划分的精英解网格，兼顾质量与多样性，避免过早收敛；② **异步 DAG 评估流水线**——把候选解的评估组织成有向无环图并行执行，提升吞吐；③ **LLM 驱动的变异算子**——带 **insight 生成**（让 LLM 解释为何这样变异）+ **双向 lineage 追踪**（记录每个解的祖先/后代谱系）；④ **灵活的 multi-island 演化策略**——多个独立演化"岛屿"并行、定期迁移。系统强调"模块化、并发、易实验"，支持声明式配置快速原型。

**关键结果**：在 AlphaEvolve 论文的三个挑战问题上复现验证实现正确性——**Heilbronn 三角放置、圆填充（circle packing）、高维 kissing number**。代码托管在 AIRI-Institute/gigaevo-core。8 位作者（Valentin Khrulkov、Ivan Oseledets 等）。

**意义**：GigaEvo 为"LLM × 进化算法"社区提供了可复现的基础设施——把 AlphaEvolve 这一里程碑工作从"知道它很强但复现不了"变成"开源可用、可改、可扩展"。它是本库演化式优化派（与 [[13-cse]] CSE、[[14-quantaalpha]] QuantaAlpha、BES、TTT-Discover 等）的工程基座，让后续研究者不必从零重建 MAP-Elites + 异步评估 + LLM 变异这套复杂机制。在"进化的单位上移"趋势里，它守住了最底层的"算法/代码进化"这一原始战场。

#### #17 Agent0: Unleashing Self-Evolving Agents from Zero Data via Tool-Integrated Reasoning 📖 [UNC, 110↑] — arXiv 2511.16043 → 精读 [[01-agent0]]〔分类①〕

**问题动机**：LLM agent 用 RL 训练时严重依赖人类策划的数据——既限制规模，也把 AI 能力上限拴死在人类知识上。已有自演化框架的两大短板：受限于模型自身能力（自己出的题不会超过自己太多）、且多为单轮交互，难构造涉及工具使用/动态推理的复杂课程。

**核心方法**：Agent0 从同一个 base LLM（Qwen3-Base）初始化两个 agent——**Curriculum Agent π_θ（出题者）**与 **Executor Agent π_φ（解题者）**——做 T=3 轮共生竞争（symbiotic competition）。每轮两阶段：① 用 GRPO 训 π_θ 生成挑战*冻结*的 executor 的任务；② 冻结 π_θ、生成任务池、过滤出数据集，再用 **ADPO**（Ambiguity-Dynamic Policy Optimization）训 π_φ。**任务过滤**靠 self-consistency p̂(x)：保留 |p̂−0.5|≤δ=0.25（即 p̂∈[0.3,0.8] 的"不太易也不太难"任务）。**Curriculum reward** 三信号：Uncertainty + Tool Use（鼓励工具调用，cap=4）+ Repetition Penalty。**工具集成推理**用"stop-and-go"多轮 rollout（生成→暂停→沙箱执行→结果回传→继续到 \boxed{}）。**工具是打破能力天花板的关键**——让 co-evolution 不早熟（出题者必须构造工具感知的更难任务）。ADPO 用 Ambiguity-Aware Advantage Scaling（降权噪声伪标签）+ Ambiguity-Modulated Trust Regions（对模糊任务放松 clip）。

**关键结果**：Qwen3-8B-Base 上**数学推理 +18%、通用推理 +24%**，全程零外部数据。数学 AVG 49.2→58.2（超 R-Zero 6.4%、Socratic-Zero 3.7%）。**共进化进程证据**：固定 executor pass rate 逐轮 64.0→58.5→51.0 下降、平均 tool call 1.65→2.10→2.60 上升。1.2k★。

**意义**：Agent0 是**零数据自演化奠基作**——证明 agent 可"自己造课程、自己出题、自己进化"，摆脱人类数据天花板。出题者-解题者对抗是 self-play 的课程版，工具集成是不早熟的关键。它与同组（aiming-lab）的 [[05-skillrl]] SkillRL、MetaClaw、Agent0-VL 一脉，是本库"co-evolution + zero-data"主线的源头，也是 agentic-rl 专题 [[agentic-rl/05-agent0]] 的同一工作。精读见 [[01-agent0]]。

#### #29 AutoEnv: Automated Environments for Measuring Cross-Environment Agent Learning 📄 [FoundationAgents, 92↑] — arXiv 2511.19304 → 见 [[16-autoenv]]〔分类④〕

**问题动机**：人类能在动态各异的世界间适应，但现有 agent 多在**单一域内**自演化，隐含"环境分布固定"的假设；**跨环境学习（cross-environment learning）几乎无人测**——我们根本不知道 agent 学到的能力能否迁移到动态不同的新环境。

**核心方法**：AutoEnv 把环境当作**"转移/观察/奖励的可因子化分布（factorizable distribution）"**——这种分解让框架能独立改变每个因子、自动生成异构世界，平均**仅 $4.12** 低成本生成一个环境。基于此构建 **AutoEnv-36**（36 个环境、358 个验证关卡）。把 agent 学习形式化为**组件中心（component-centric）的过程**，由 **Selection（选择改进哪个组件）- Optimization（优化）- Evaluation（评估）** 三阶段驱动，据此设计 **8 种学习方法**。

**关键结果**：跨 **7 个 LM**，归一化奖励仅 **12–49%**——跨环境学习是真实挑战。关键发现：**任何单一学习方法的增益随环境数增多而快速衰减**（固定方法不跨异构世界 scale）；**环境自适应地选择学习方法**能大幅提升，但随方法空间扩大呈现边际递减。代码 github.com/FoundationAgents/AutoEnv。15 作者。

**意义**：AutoEnv 把"跨环境学习"这一被忽视的盲区建成可量化的测试床——它揭示了**现有自演化方法在单域内有效、但跨异构环境几乎不 work**（7 个 LM 仅 12-49%）。这直接定义了本库"终身、跨域演化"未来方向的起点（见趋势 方向1️⃣）。它把"环境"从背景提升为一等公民，与 [[03-agent-world]]（环境自合成）、[[18-evoarena]]（环境演化评测）、[[22-agentic-env-survey]]（环境工程综述）共同构成"环境驱动能力进化"主线。

#### #33 Scaling Agent Learning via Experience Synthesis (DreamGym) 📖 [Meta, 83↑] — arXiv 2511.03773 → 精读 [[04-dreamgym]]〔分类①〕

**问题动机**：RL 能通过交互增强 LLM agent，但实际落地受阻于**昂贵的 rollout、有限的任务多样性、不可靠的奖励信号、复杂的基建**——这些都阻碍可扩展经验数据的收集。

**核心方法**：DreamGym 是**首个为可扩展性设计、用合成经验做 online agent RL** 的统一框架。核心是**把环境动态蒸馏成一个 reasoning-based experience model（基于推理的经验模型）**——给定 `(state, action)`，模型通过 step-by-step 推理产生**一致的状态转移与反馈信号**，替代昂贵的真实 rollout（用语言推理刻画状态转移逻辑，一次 LLM 前向代替一次容器执行）。三件套：reasoning-based experience model + **experience replay buffer**（用真实数据初始化、持续补充新交互）+ **自适应任务生成**（在线课程，生成挑战当前 policy 的新任务）。对照 RL 算法用 GRPO 与 PPO，核心卖点是"用什么数据喂 RL"。

**关键结果**：WebArena 非 RL-ready 设定超所有基线 **30%+**；RL-ready 但 rollout 昂贵的设定下纯合成交互即**媲美 GRPO/PPO**；**sim-to-real warm-start** 用远少于真实交互量获得显著额外增益。作者含 Jason Weston、Dawn Song、Bo Li。

**意义**：DreamGym 把 agent RL 的成本从"维护真实环境集群"降到"训练 + 调用经验模型"，确立 world-model-for-RL 在 LLM agent 语境下的可行性（用推理而非像素生成）。它的"reasoning-based experience model"本质是轻量 world model，是本库"经验合成 / 自造数据"主线的源头（与 [[03-agent-world]] 合成环境、[[02-evocua]] 合成 CUA 经验呼应），也是 agentic-rl 专题 [[agentic-rl/01-dreamgym]] 的同一工作。精读见 [[04-dreamgym]]。

#### #69 Agent0-VL: Exploring Self-Evolving Agent for Tool-Integrated Vision-Language Reasoning 📄 [UNC, 49↑] — arXiv 2511.19900〔分类⑦〕

**问题动机**：视觉-语言 agent 的学习受限于人类标注监督。已有 self-rewarding 方法让模型当自己的 critic，但**纯文本的 self-evaluation 无法验证复杂的视觉推理步骤、且常产生评估幻觉**——模型"看错了还以为自己对"。

**核心方法**：Agent0-VL（[[01-agent0]] 的视觉版）的关键创新是**把工具集成不只用于推理，还用于 self-evaluation 与 self-repair**。它在单个 LVLM 里统一两个角色：**Solver**（做多轮工具集成推理）+ **Verifier**（通过 tool-grounded critique 产生结构化反馈 + 细粒度 self-reward）。**缓解视觉自评幻觉**的办法是——把验证 grounding 在工具证据上而非纯文本内省（让 agent 用工具实际核验自己的视觉推理结论，而非凭感觉自判）。两角色通过 **Self-Evolving Reasoning Cycle** 交互，tool-based verification 与 RL 联合对齐推理与评估分布，实现"zero-external-reward evolution"（无需人工标注或外部奖励模型）。

**关键结果**：在几何问题求解 + 视觉科学分析上评测，**比 base model +12.5%**。代码 github.com/aiming-lab/Agent0。

**意义**：Agent0-VL 把自演化推进到 VLM，针对性地解决了"视觉自评幻觉"这一多模态特有难题——**用工具证据 grounding 自我验证**，让 self-evolution × tool-integrated × multimodal 三者交叉可行。它是本库"自演化向多模态扩散"主线（与 VisPlay、MM-Zero、UniCorn、SpatialEvo）的早期代表，证明 [[01-agent0]] 的零数据自演化范式可移植到视觉模态、且工具集成是缓解视觉幻觉的钥匙。57★。

#### #75 VisPlay: Self-Evolving Vision-Language Models from Images 📄 [WashU/Maryland, 45↑] — arXiv 2511.15661〔分类⑦〕

**问题动机**：RL 是改进 VLM 复杂推理的有原则框架，但现有方法依赖人工标签或任务专属 heuristics 来定义可验证奖励——**成本高、难 scale**。能否从海量无标注图像自演化 VLM？

**核心方法**：VisPlay 是**从无标注图像自演化的 RL 框架**。从单个 base VLM 出发，切成两个交互角色：**Image-Conditioned Questioner**（出"有挑战但可答"的视觉问题）+ **Multimodal Reasoner**（产 silver 答案）。两角色用 **GRPO 联合训练**，GRPO 整合 **diversity 奖励**（问题多样性）和 **difficulty 奖励**（问题难度），平衡"生成问题的复杂度"与"silver 答案的质量"。关键是**操作在无标注图像数据上**，移除了对人工标注标签的依赖。

**关键结果**：在 Qwen2.5-VL 和 MiMo-VL 两个模型族、8 个基准（含 MM-Vet、MMMU）上评测，在视觉推理、组合泛化、幻觉减少上均有提升。项目页公开。57★。

**意义**：VisPlay 把 [[01-agent0]] 式双角色自博弈落地到视觉模态，规模化利用海量无标注图像——证明 VLM 也能像 LLM 一样"从无标注数据自演化"。它与 Agent0-VL（工具增强自评）、MM-Zero（三角色零数据）、UniCorn（理解→生成）共同把自演化范式推广到多模态，各自解决模态特有难题（VisPlay 解决"如何从无标注图像定义可验证奖励"）。

#### #58 Guided Self-Evolving LLMs with Minimal Human Supervision (R-Few) 📄 [腾讯, 55↑] — arXiv 2512.02472〔分类①〕

**问题动机**：自演化被设想为通往超级智能的路径，但**无引导的自演化系统常快速 plateau 甚至退化**。根因是三个失败模式——模型"强化自身偏见、收敛到低熵行为"：**concept drift**（迭代中逐渐失去与任务概念的对齐）、**diversity collapse**（输出多样性萎缩）、**mis-evolution**（强化内部偏见而非纠正）。

**核心方法**：**R-Few** 是 **guided Self-Play Challenger-Solver 框架**。每轮：**Challenger** 采样一小组人类标注样本来引导合成问题生成；**Solver** 在"在线、难度递进的课程"下联合训练人类样本 + 合成样本。两个关键技术：① **in-context grounding**——通过标注样本注入轻量人类监督来引导生成；② **mixed training**——混合人类与合成数据稳定 co-evolution。即用极少量人类监督把自演化稳定可控。

**关键结果**：Qwen3-8B-Base 上比 R-Zero **+3.0 分**（数学）；与 General-Reasoner 持平，**而后者用了 20× 更多人类数据**。Ablation 证实 grounded challenger 训练与课程化 solver 训练的互补贡献，R-Few 缓解 drift、产生更稳定可控的共进化动态。作者含 Haitao Mi、Dong Yu（腾讯）。

**意义**：R-Few 是**自演化稳定性问题的代表性诊断 + 解法**——它明确命名了无引导自演化的三种塌缩模式（concept drift / diversity collapse / mis-evolution），并用"极少人类监督 + 课程 + 混合训练"稳住。这是本库"防对抗崩溃/防塌缩"主线（与 [[08-ctx2skill]] Cross-time Replay、OpenSkill/SCOPE 开放世界 bootstrap）的关键一环，直接对应趋势 方向🔟"开放世界 bootstrap + 防塌缩"。它证明：完全无引导的自演化会退化，少量 grounding 就能换来巨大稳定性（vs 20× 数据的强基线）。

#### #96 Reinforcement Learning for Self-Improving Agent with Skill Library (SAGE) 📄 [42↑] — arXiv 2512.17102〔分类②〕

**问题动机**：LLM agent 虽在复杂推理与多轮交互上能力强，但**部署到新环境时难以持续改进与适应**。现有 skill library 方法**多靠 LLM prompting，实现不稳定**（一致性差）——用 RL 系统性地把 skill 纳入学习是更可靠的路线。

**核心方法**：**SAGE（Skill Augmented GRPO for self-Evolution）** 把 skill 系统性融入 GRPO 学习，两个关键机制：① **Sequential Rollout**——每次 rollout 让 agent **在一连串相似任务上迭代部署**，早期任务产生的 skill **累积进库、供后续任务使用**（边解边攒 skill）；② **Skill-integrated Reward**——一个补充原始 outcome-based 奖励的信号，同时强化 skill 生成与 skill 利用。两者驱动 GRPO 更新。

**关键结果**：在 **AppWorld** 上，SAGE 应用到带专家经验 SFT 的模型时——**Scenario Goal Completion 高 8.9%、交互步数少 26%、生成 token 少 59%**，在准确率与效率上都大幅超过现有方法。

**意义**：SAGE 是 [[05-skillrl]] SkillRL / [[06-skill1]] Skill1 同主题的**早期工作**（skill 库与 RL 结合），强调"一致的 skill 实现"——它指出纯 prompting 式 skill 库实现不稳定、改用 RL（SAGE=Skill-Augmented GRPO）系统化。它属于本库 Skill 进化的"神经 RL 派"，其"Sequential Rollout 让 skill 跨相似任务累积"的设计，与 SkillRL 的递归进化、Skill1 的统一三能力一脉，共同探索"如何用 RL 让 skill 库与 policy 共进化"。

#### #4 Agentic Reasoning for Large Language Models 📖 [UIUC, 206↑] — arXiv 2601.12538 → 精读 [[21-agentic-reasoning-survey]]〔分类⑧〕

**问题动机**：agentic reasoning 领域快速膨胀但缺乏统一的体系化梳理——各种工作（ReAct、ToT、o1、Deep Research、self-evolving agent...）散落各处，缺一份按环境动态组织的、能定位所有工作的框架。

**问题动机**：RL 在可验证 outcome 任务上提升显著，但在**开放式 agent 任务**（如复杂旅行规划，解空间巨大、无客观 ground-truth）上挣扎。因缺客观答案，现有 RL 靠 reward model 给单个响应打**标量分**——但 pointwise scoring 有**discrimination collapse**：reward model 难分辨轨迹间的细微优势，组内分数被压缩到窄区间，有效奖励信号被 reward model 噪声淹没，优化停滞。

**关键结果**：种子单淘汰以 O(N) 达到接近 O(N²) 完整 pairwise 的精度；ArenaRL 大幅超标准 RL 基线，让 agent 在复杂真实任务上产出更鲁棒的解。254★。

**意义**：ArenaRL 为"开放式、无标准答案"任务的演化式 RL 提供**可扩展的相对评价信号**——用锦标赛相对排名替代标量打分，破解 discrimination collapse。它与 SCOPE（自判 rubric）、OpenSkill（自造 verifier）共同探索"开放式任务如何获得可靠奖励信号"，是本库"开放世界 bootstrap"主线在"评价信号"侧的代表。其 O(N) 锦标赛 vs O(N²) pairwise 的效率权衡也呼应 方向8️⃣"让自演化用得起"。

**问题动机**：持久 skill 库让 agent 跨任务复用成功策略，但维护它需要三个**耦合**能力——选择相关 skill、执行时利用 skill、从经验蒸馏新 skill。如何用**一个 RL 框架统一**这三者？

**核心方法**：**Skill1** 用**单一 policy + 单一 task-outcome 信号**统一协同进化 skill 的**选择/使用/蒸馏**三能力。流程：agent 生成查询搜 skill 库 → 重排选最相关 → 条件求解 → 成功后蒸馏新技能加库。核心：从单一信号按**频率分解** credit——**低频趋势归因"选择"**（找对 skill 就赢的低频任务）、**高频变化归因"蒸馏"**（创造新 skill 解锁的高频任务）。把整个 skill lifecycle 用一个 reward 信号统一驱动。

**关键结果**：ALFWorld / WebShop 超 skill-based 与 RL 基线；训练动态证实三能力 **co-evolve**。

**意义**：Skill1 避免"skill selector/executor/extractor 各训各的"碎片化——把 skill lifecycle 用一个 reward 统一。它的"按频率分解 credit"（低频→选择、高频→蒸馏）是优雅的信用分配设计。它是 Skill 进化神经 RL 派代表，与 [[05-skillrl]]（递归进化）、SAGE 一脉。它也是 agentic-rl 专题 [[agentic-rl/00-summary]] 的同一工作。精读见 [[06-skill1]]。

#### #78 Co-Evolving Policy Distillation (CoPD) 📄 [67↑] — arXiv 2604.27083〔分类③〕

**问题动机**：把多个专家能力整合进单一模型，RLVR 与 OPD（On-Policy Distillation）两条路各有能力损失：**mixed RLVR** 有"inter-capability divergence cost"（能力间分歧代价）；**先训专家再 OPD** 避开了分歧，但因**师生行为模式差距大**而"无法完全吸收 teacher 能力"。

**核心方法**：**CoPD** 的核心：① **并行专家训练**（专家并发训而非串行）；② **在持续 RLVR 过程中就引入 OPD**（distillation 发生在每个专家**正在进行的 RLVR 训练中**，而非训完之后）；③ **双向 OPD**——专家**互为师生**，distillation 双向流动实现 co-evolve；结果是在保留互补知识的同时保持行为模式一致。

**关键结果**：实现 **text、image、video 推理的 all-in-one 整合**，显著超过 mixed RLVR、MOPD 等强基线，甚至超过领域专家模型。作者认为并行训练模式"可能启发新的训练 scaling 范式"。

**意义**：CoPD 把"co-evolution"用于**多专家能力的单模型整合**——避免能力损失。它的创新是"在 RLVR 过程中就双向 OPD"（专家互为师生、边训边蒸），而非"先训完再单向蒸馏"。它属本库演化式优化/共进化分类，与本库 co-evolution 主线（出题↔解题、skill↔policy、agent↔环境）一脉，但对象是"多个专家能力"。它对多模态统一模型（text/image/video）的整合有直接价值，呼应趋势 方向3️⃣"co-evolution 取代单向训练"。

#### #88 Self-Improving Language Models with Bidirectional Evolutionary Search (BES) 📄 [Harvard, 59↑] — arXiv 2605.28814〔分类③〕

**问题动机**：search 是自改进 LM 的有效方法，但 best-of-N 和树搜索有两个根本局限：① **受稀疏验证信号引导**；② **主要靠自回归扩展构造候选**，把探索局限在高概率质量区域（model probability mass）。

**核心方法**：**BES（Bidirectional Evolutionary Search）** 耦合**前向候选进化**与**后向目标分解**。**前向搜索**——在标准扩展之上加**进化算子重组部分轨迹**，生成"单次 model rollout 难以获得的候选"（跳出高概率区）；**后向搜索**——**递归把原任务分解为可检查的子目标（checkable subgoals）**，产生密集中间反馈引导前向搜索。理论动机：扩展-only 候选被困在"窄熵壳（narrow entropy shell）"，进化算子能逃逸；后向搜索能**指数级减少找到正确答案所需的样本数**。

**关键结果**：在主流后训练算法无法提升的挑战性后训练任务上，BES 实现一致增益；在 3 个开放问题求解基准（推理时），BES 在平均和最佳性能上都超过现有开源框架。153★。

**意义**：BES 把**进化算子注入 LLM 自改进搜索**，突破自回归扩展的探索局限（窄熵壳）——这是它对 best-of-N/树搜索两大局限（稀疏信号 + 高概率区局限）的针对性破解。它属本库演化式优化派，与 [[13-cse]]、[[15-gigaevo]]、TTT-Discover 共享"进化算子"思想，但独特在于"前向进化 + 后向分解"双向耦合 + 理论分析（熵壳逃逸、指数样本减少）。它呼应趋势 方向5️⃣"自演化理论化"（给出可证明的样本复杂度结果）。

#### #14 EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments 📖 [NUS等, 90↑] — arXiv 2606.13681 → 精读 [[18-evoarena]]〔分类⑤〕

**问题动机**：现有 agent 评测多假设**静态环境**——但真实部署中环境**渐进变化**（软件更新、社交规则演变、终端状态迁移）。agent 的记忆必须能记录并适应这种演化，否则会用过时记忆做错误决策。

**核心方法**：**EvoArena** 把环境变化建模为 **terminal/software/social 三域的渐进 update 序列**（一系列对环境的结构化改动）——这是**首个把"环境随时间演化"作为核心评测维度**的基准。配套 **EvoMem**——**patch-based 记忆**，以**结构化 update history** 记录记忆演化（每次环境变化作为一个 patch 追加），让 agent 通过"记忆变化"推理"环境演化"。

**关键结果**：现有 agent 在 EvoArena 仅 **39.6%**；EvoMem 在 **GAIA +6.1%、LoCoMo +4.8%、chain-level +3.7%**。

**意义**：EvoArena 指出了被忽视的评测盲区——**真实环境是动态的，记忆必须记录演化**。它把 working memory 研究从"静态环境管理上下文"推进到"动态环境追踪并适应变化"。它与 `memory/` 专题同一工作交叉（[[memory/08-evoarena-evomem]]）。它是本库"环境从背景变一等公民"（方向7️⃣）与"动态评测"（方向7️⃣评测侧）的代表，证明现有 agent 在演化环境上仅 39.6%——静态基准已不足以衡量。精读见 [[18-evoarena]]。

#### #34 Agentic Environment Engineering for LLMs: A Survey 📖 [中科院, 58↑] — arXiv 2606.12191 → 精读 [[22-agentic-env-survey]]〔分类⑧〕

**问题动机**：环境是 LLM agent 跨场景的交互系统，对驱动模型能力持续进化至关重要，但**现有工作缺乏系统性分类与深度分析**。

**核心方法**：本综述（中科院，15 作者，63 页）通过**环境工程生命周期**透镜研究该领域——**Modeling（用 8 属性 × 8 领域刻画代表性环境）→ Synthesis（symbolic 与 neural 两种自动合成范式）→ Evaluation（与合成范式对应的评估方法）→ Application（通过 agent-environment co-evolution）**。提出 **agent 进化四路径**：**memory-centric（经验进化）/ orchestration-centric（工作流进化）/ trajectory-centric（离线进化）/ exploration-centric（在线进化）**——几乎完美覆盖本库所有论文。**环境演化三范式**：**neural-driven / difficulty-driven / scaling-driven**。明确的 open problems：**Environment-as-a-Service（EaaS）、Multi-agent Environments、Neural-Symbolic Environments**。

**关键结果**：提供环境工程的完整生命周期框架 + 四进化路径 + 三演化范式 + 未来方向。

**意义**：本综述是**本库的学术坐标系之二**（环境视角）——与 [[21-agentic-reasoning-survey]]（agent 视角）互补，共同为整个 evolve 主题定调。它的**四进化路径**（memory/orchestration/trajectory/exploration-centric）和**三环境演化范式**（neural/difficulty/scaling-driven）是本库分类与趋势分析的权威依据。它明确提出的 **EaaS、Neural-Symbolic、Multi-agent Environments** 正是本库趋势 方向3️⃣/4️⃣的来源。精读见 [[22-agentic-env-survey]]。

#### #66 Task-Focused Memorization for Multimodal Agents (TaskMem) 📄 [字节 Seed, 38↑] — arXiv 2605.31075〔分类⑤〕

**问题动机**：多模态 agent（如具身 agent）持续感知/推理/行动，面对**无界的多模态观察流**——核心挑战不是记忆模块设计，而是**determine what to memorize（决定该记什么）**：从信息的组合爆炸中选择性保留"与角色相关、对未来任务有价值"的内容。

**核心方法**：**TaskMem** 把记忆生成**重构为可学习的 memorization policy（记忆策略）**，是 RL 框架，让策略**动态调整关注点**匹配真实任务需求。两阶段：① **Phase One（学怎么记）**——基本保真度下优化记忆质量；② **Phase Two（学该记什么，部署后）**——在 base MLLM 上 tune adapter，用**近期环境任务定义 reward model** 引导记忆策略偏向任务相关内容。评测把 VideoMME/EgoLife/EgoTempo 重构成流式基准（问题只能用记忆答、不能看原始视频）。

**关键结果**：基于 Qwen3-VL-30B-A3B，三个基准 VQA 准确率 **+6.3% / +7.0% / +5.3%**。

**意义**：TaskMem 把"记忆 = 可学习策略"用于多模态持续学习——它明确把"该记什么"建模成可学习的 memorization policy，"部署后用近期任务定义 reward"实现任务自适应。它与 [[02-evocua]]（字节）同源工业线，与 `memory/` 专题同一工作交叉（[[memory/00-summary]] E 类），与 MemSkill（记忆 skill 进化）共同代表本库"记忆进化"分类。它把记忆写入策略由 RL 学习并扩展到多模态/具身场景，呼应"进化单位上移到记忆"（方向6️⃣）。

#### #70 SkillAdaptor: Self-Adapting Skills for LLM Agents from Trajectories 📄 [蚂蚁集团, 36↑] — arXiv 2606.01311〔分类②〕

**问题动机**：现有 training-free skill 适配从**整条轨迹/会话级反馈**更新 skill，**失败归因太粗**——产生不稳定、过宽的修订（明明只是某一步错，却把整条经验都改了）。

**核心方法**：**SkillAdaptor** 是 **training-free 的 step 级 skill 自适应**框架，带显式失败归因。给定失败轨迹：① **定位"首个可操作的故障步"（first actionable fault step）**；② 把责任**链接到候选 skill**；③ 在**显式接受检查（acceptance checks）下做定向更新**；④ **保持 backbone 冻结**。可插入 OpenClaw 类 agent harness。

**关键结果**：在 WebShop / PinchBench / Claw-Eval 上用 Kimi-K2.5 / GLM-5 / GPT-5.2 评测，超过 no-skill 与 skill-adaptation 基线——PinchBench +1.5、Claw-Eval +1.8、WebShop +1.7。代码 github.com/zjunlp/SkillAdaptor。

**意义**：SkillAdaptor 把 [[09-skillsvote]] 的"精细归因 + 门控"做到 **step 级**（定位首个可操作故障步），frozen backbone。它解决了"整轨迹级归因太粗导致过宽修订"问题——step 级归因支持更稳定、可审计的 training-free skill 维护。它与 SkillsVote（生命周期治理）、SkillOpt（text-space 优化）共同代表"frozen model + 外部 skill 精细进化"，是本库"防污染/精细归因"主线在 step 粒度的推进，可插 OpenClaw harness 也呼应 agentic-rl 的 OpenClaw-RL。

#### #90 Language Models Need Sleep: Learning to Self-Modify and Consolidate Memories 📄 [Google, 29↑] — arXiv 2606.03979〔分类⑤〕

**问题动机**：现有模型缺乏持续学习、缺乏把临时 in-context 知识迁移到长期参数的能力。人类靠睡眠把短期记忆巩固成长期记忆——LLM 能否也有"睡眠"阶段？

**核心方法**：受人类学习启发，提出 **"Sleep" 范式**让模型 **self-modify**：① **Memory Consolidation**——把脆弱的**短期记忆**经 **replay 蒸馏进稳定的长期参数知识**（其中 Knowledge Seeding 是"小自我→大网络"的向上蒸馏）；② **"Dreaming"**——递归自我改进（RL 自生成课程自我精进）。整个过程是 agent 在"清醒"交互之外的"离线巩固"阶段。

**关键结果**：作为 proof-of-concept，展示短期记忆向长期参数巩固的可行性。29★。

**意义**：Sleep 范式触及深刻问题——**短期记忆（in-context）与长期记忆（参数）之间的巩固边界**。它把"睡眠巩固"这一生物学机制引入 LLM，是 [[03-agent-world]] DreamGym（reasoning-based experience model）之外又一"梦境/巩固"思路（Google 出品）。它与 `memory/` 专题同一工作交叉（[[memory/00-summary]] E 类）。它属本库"记忆进化"分类，"经验→权重巩固 + 自我修改"也呼应 方向9️⃣"递归/元自改进"（Dreaming 是自生成课程的自我精进）。虽仍是 proof-of-concept，但指向"经验如何沉淀为长期参数"的根本问题。

#### #94 SCOPE: Self-Play via Co-Evolving Policies for Open-Ended Tasks 📄 [28↑] — arXiv 2605.31433〔分类①〕

**问题动机**：self-play 能无外部监督训练 LM，但**现有方法需可规则核验的答案**——开放式任务只能靠 curated prompt 或 frontier judge（贵且依赖外部）。如何在开放式任务上做**无数据 self-play**？

**核心方法**：**SCOPE** 是开放式任务的**data-free self-play 框架**，co-evolve 两个 policy：**Challenger**（产**文档锚定（document-grounded）的任务**）+ **Solver**（多轮检索作答）。关键是用**初始模型的冻结副本作 self-judge**——它**从源文档写 task-specific rubric 并据此给 Solver 打分**（无需 curated 答案或 frontier judge，故 data-free）。co-evolve Challenger 是必要的——让任务保持在 Solver 能力边缘。

**关键结果**：3 个 7-8B 模型（Qwen2.5/Qwen3/OLMo-3）开放式性能（8 基准）最高 **+10.4 分**，匹敌/超过用 ~9K curated 数据训的 GRPO；虽只在开放式任务训，held-out 短答案 QA（7 基准）也最高 **+13.8 分**，在所有三个模型上超过 GRPO。Ablation：co-evolve Challenger 必要、增益来自检索与合成两者、rubric 生成质量是自判瓶颈。

**意义**：SCOPE 把自演化推到"**开放式 + 无数据**"——它的创新是用"初始模型冻结副本从源文档写 rubric 自判"绕过"开放式任务需 curated 答案/frontier judge"的障碍。它与 [[08-ctx2skill]]（三方自博弈）、UniCorn（Proposer-Solver-Judge）、ArenaRL（相对排名）共同探索"开放式任务如何获得可靠奖励信号"。它证明 data-free self-play 能匹敌用 9K 数据的 GRPO（甚至迁移到短答案 QA），是本库"开放世界 bootstrap"主线（方向🔟）的代表。

#### #95 OpenSkill: Open-World Self-Evolution for LLM Agents 📄 [OpenLAIR, 27↑] — arXiv 2606.06741〔分类②〕

**问题动机**：现有自演化假设有可用的学习循环（curated skill / 成功轨迹 / verifier 信号），但真实开放世界**可能什么都没有**——只给一个任务 prompt。如何在"无 verifier、无成功轨迹"的最难场景自演化？

**核心方法**：**OpenSkill** 研究 **open-world self-evolution**——agent 必须**从零自建 skill 与验证信号**，用开放世界资源但**无目标任务监督**。三阶段：① **从文档/仓库/web 获取 grounded 知识 + 验证锚点（verification anchors）**；② **合成可迁移 skill**；③ 在**自建虚拟任务**上精炼，这些任务**锚定在 anchor 而非目标答案上**。关键洞察：开放世界同时提供"要学的知识"和"监督无关的练习环境"，目标任务监督只留作最终评估。

**关键结果**：3 个基准 × 2 个目标 agent，在满足无监督约束下达**最佳自动 pass rate**；skill 跨模型迁移无需模型特定适配；**自建 verifier 与 ground-truth 结果对齐**（尽管从未访问 ground truth）。代码 github.com/OpenLAIR/OpenSkill。20 页。

**意义**：OpenSkill 把自演化推到"**无监督、无 verifier**"的最难场景——open-world bootstrap。它的创新是 agent **从 web/文档/仓库自建验证锚点**（而非依赖给定 verifier），且自建 verifier 竟与真实结果对齐。它是本库"zero-data / open-world bootstrap"主线（趋势 方向1️⃣"自造数据推到极限"、方向🔟）的最激进代表——连验证信号都自己造。它与 SCOPE（自判 rubric）、ArenaRL（相对排名）共同回答"开放世界如何自造学习循环"，但 OpenSkill 设定最极端（什么都没有）。48★。

### 3️⃣ Co-Evolution（协同进化）取代单向训练

- [[01-agent0]] curriculum↔executor、[[05-skillrl]] skill库↔policy、[[03-agent-world]] agent↔环境、[[08-ctx2skill]] 出题↔解题
- 共性：**两个主体互相驱动、互相施压**——本质是"GAN 思路"在 agent 训练的各种变体

### 4️⃣ "防对抗崩溃 / 防污染"成为自演化系统的标配模块

- [[08-ctx2skill]] Cross-time Replay、[[09-skillsvote]] evidence-gated update、[[07-psn]] maturity-aware gating + rollback
- 自演化系统的稳定性问题被正面解决——这是从"能演化"到"稳定演化"的成熟标志

### 5️⃣ LLM × 进化算法（遗传操作）持续升温

- [[13-cse]] CSE（代码）、[[14-quantaalpha]] QuantaAlpha（金融）、[[15-gigaevo]] GigaEvo（AlphaEvolve 开源）、BES（双向进化搜索）、TTT-Discover（测试时进化发现）、CoPD（共进化蒸馏）
- 共性：把**变异/交叉**从盲目随机升级为**反馈引导的受控演化**；轨迹级操作 + 经验复用

### 6️⃣ 进化的"单位"在不断上移

- 单能力（推理）→ skill → **工具**（Test-Time Tool Evolution）→ 环境/课程 → **记忆**（[[18-evoarena]]、MemSkill）→ **组织**（[[19-onemancompany]]）→ **agent 设计 agent**（Memento-Skills）→ **自指元进化**（Hyperagents）
- 从"一个 agent 变强"扩展到"整个 agent 生态/组织、乃至 agent 的学习机制本身随经验变强"

### 7️⃣ 环境从"背景"变成"一等公民"

- [[03-agent-world]] 环境自合成、[[16-autoenv]] 跨环境学习、[[18-evoarena]] 环境演化评测、[[22-agentic-env-survey]] 环境工程生命周期、SWE-Universe（百万级可验证环境）、SpatialEvo（确定性几何环境）
- 共识：**环境是驱动能力持续进化的关键**——不能只优化模型

### 8️⃣ 两篇综述定调（agent 视角 + 环境视角），第三篇补"外部化"视角

- [[21-agentic-reasoning-survey]]（UIUC，1月）：能力分层框架，确立 self-evolving layer
- [[22-agentic-env-survey]]（中科院，6月）：环境生命周期 + 四条进化路径
- Externalization 综述（上交，4月）：memory/skill/protocol/harness 的"外部化"统一视角
- 多篇综述为整个 evolve 主题提供完整学术坐标系

### 9️⃣ Frozen Model + 外部状态进化

- 闭源 frontier 模型时代，权重不可得 → 进化外部 skill / 记忆 / 课程 / 工具
- [[09-skillsvote]]、HF SkillOpt、SkillAdaptor、OpenSkill、Externalization 综述 证明：**不更新权重也能让 frozen agent 持续变强**

### 🔟 "失败"成为被重视的养料

- [[02-evocua]] 失败轨迹转富监督、[[13-cse]] 记成功+失败经验、[[20-proda]] 失败驱动数据修复、UI-Voyager（从失败经验学习）、SkillAdaptor（首个可操作故障步定位）
- 演化系统不再丢弃失败——失败包含"能力边界"的最有价值信息

### 1️⃣1️⃣ 自演化向多模态 / 空间 / GUI 全面扩散（top-100 新增主线）

- VLM：Agent0-VL、VisPlay、MM-Zero、UniCorn；空间：SpatialEvo；GUI：UI-Voyager、Step-GUI
- 共性：把文本域验证成熟的自演化范式（双/多角色 self-play、零数据）**移植到新模态**，并各自解决模态特有难题（如 SpatialEvo 用确定性几何破伪标签自我强化）

### 1️⃣2️⃣ 元（meta）/ 自指自改进成为新前沿

- Hyperagents（Darwin Gödel Machine 思路，task agent + meta agent 自指）、Memento-Skills（agent 设计 agent）、Language Models Need Sleep（自我修改 + 巩固）
- 共性：不只改进"解题能力"，而是改进"**改进能力本身**"——递归自我改进（recursive self-improvement）的早期探索

## 三、研究发展现状与方向趋势

### 3.1 发展现状：八个月走过的三个阶段（top-100 视角，50 篇）

把 2025-11 ~ 2026-06 的 **50 篇**按时间铺开，"agent evolve"主题清晰地走过了**三个阶段**：

```
   2025-11 ────────── 2026-01/02 ────────── 2026-03/04 ────────── 2026-05/06
   ① 萌芽/单点突破        ② 范式确立+扩散          ③ 体系化/治理/元进化/最新前沿
   ─────────────       ─────────────         ─────────────────────
   • Agent0 零数据       • Agentic Reasoning    • SkillsVote skill 治理
   • DreamGym 经验合成     综述确立 self-         • Skill1 统一 credit
   • GigaEvo 开源基建      evolving layer        • EvoArena 记忆/环境演化评测
   • AutoEnv 跨环境       • SkillRL/PSN/CSE/      • Hyperagents 自指元进化
     学习基准              QuantaAlpha 各流派     • Memento 设计 agent
   • Agent0-VL/VisPlay     • 向多模态扩散          • Need Sleep 自我修改
     自演化扩散到 VLM        (MM-Zero/UniCorn)     • OpenSkill 开放世界 bootstrap
                          • R-Few 诊断自演化塌缩   • Agentic Env 综述收束
```

- **阶段①（2025-11~12）单点突破 + 初步扩散**：[[01-agent0]]、[[04-dreamgym]] 证明"零数据/合成经验自演化"可行；同期 Agent0-VL/VisPlay 已把范式扩到 VLM；12 月 R-Few 率先诊断"自演化塌缩"。
- **阶段②（2026-01~02）范式确立 + 多模态扩散**：[[21-agentic-reasoning-survey]] 综述确立 self-evolving layer；三大流派代表作密集出现；MM-Zero/UniCorn 把零数据自演化推广到多模态；SWE-Universe 把环境推到百万级。
- **阶段③（2026-03~06）体系化 + 治理 + 元进化 + 最新前沿**：关键词从"能不能进化"转向"如何**稳定、可治理、可递归**地进化"——[[09-skillsvote]] 治理、[[08-ctx2skill]] 防崩溃、Hyperagents/Memento 元进化、Need Sleep 自我修改、OpenSkill 开放世界 bootstrap、[[22-agentic-env-survey]] 综述收束。

**当前成熟度判断**：

- **已解决（从研究角度）**：① 零数据/合成数据自演化的**可行性**（含多模态）；② skill 作为能力载体的**三种实现路径**；③ co-evolution 的**基本稳定性**（防崩溃/防污染门控）；④ 自演化向 VLM/空间/GUI 的**可移植性**。
- **正在解决**：① 自演化的**可治理性**（[[09-skillsvote]]、R-Few）；② 动态环境下的**记忆/评测**（[[18-evoarena]]、MemSkill）；③ 进化单位向**组织级 / agent 设计 agent / 元进化**扩展（[[19-onemancompany]]、Memento、Hyperagents）；④ **开放世界无 verifier** 的 bootstrap（OpenSkill、SCOPE）。
- **尚未解决**：见下 3.3。

| **能力载体** | skill 成为主流载体，三流派（神经/程序化/文档）并立；并扩展到工具、记忆       | [[05-skillrl]] [[07-psn]] [[09-skillsvote]] MemSkill       |

### 3.3 未来方向趋势（基于 50 篇的缺口与多篇综述的展望）

> 综合本库论文的局限（见各篇第 4 节）+ [[21-agentic-reasoning-survey]] / [[22-agentic-env-survey]] 等综述明确提出的 open problems。

**方向 1️⃣：从"单任务/单域演化"走向"终身、跨域演化（Lifelong & Cross-domain）"**

- 现状：多数工作仍在单域（ALFWorld/WebShop/OSWorld）内演化；[[16-autoenv]] 已揭示**跨环境学习几乎未被解决**（7 个 LM 只拿 12-49%）。
- 趋势：真正的 lifelong learning——agent 在持续变化的多域环境里不遗忘、可迁移。[[18-evoarena]] 的"演化链评测"是第一步。

**方向 2️⃣：自演化的"可治理性 / 安全性 / 对齐"成为必答题**

- 现状：[[09-skillsvote]] 已把"治理"独立出来；[[21-agentic-reasoning-survey]] 明确把 **governance** 列为未来方向。
- 趋势：当 agent 能自主进化能力，**如何防止能力漂移、目标偏移、skill 库被恶意/低质内容污染**会成为核心议题——evidence-gated / validation-gate 只是开始。

**方向 3️⃣：进化"单位"继续上移——从组织到多组织生态**

- 现状：进化单位已从 能力→skill→环境→记忆→**组织**（[[19-onemancompany]]）。
- 趋势：[[22-agentic-env-survey]] 提出 **Multi-agent Environments**；下一步是**多 agent 组织/市场级别的协同进化**——agent 之间形成分工、交易、竞争的演化生态。

**方向 4️⃣：Environment-as-a-Service（环境即服务）+ Neural-Symbolic 环境**

- 现状：[[03-agent-world]] 用 MCP 接入真实环境已是雏形；[[22-agentic-env-survey]] 明确提出 EaaS 与 Neural-Symbolic 环境。
- 趋势：环境像云服务一样**可按需合成、计费、复用**；符号（可验证）与神经（可生成）环境融合，兼顾保真度与多样性。

**方向 5️⃣：自演化的"理论化"——可证明的收敛性与稳定性**

- 现状：[[07-psn]] 发现"skill 进化 ≈ 神经网络训练"的同构，HF SkillOpt 把 SGD 搬到文本空间——都还是**经验类比**，缺乏理论保证。
- 趋势：把文本/符号空间的演化与优化理论对接，给出**收敛性、稳定性、样本复杂度**的可证明结果——这是该领域从"工程"走向"科学"的关键。

**方向 6️⃣：World Model 与自演化深度融合**

- 现状：[[04-dreamgym]] 的"reasoning-based experience model"本质是轻量 world model；[[21-agentic-reasoning-survey]] 把 **world modeling** 列为深水区。
- 趋势：用更强的 world model 生成更真实的"想象经验"，让 agent 在"脑内沙盘"里大规模演化——sim-to-real 的保真度是关键瓶颈。

**方向 7️⃣：评测体系跟上——动态、演化、经济价值导向**

- 现状：[[18-evoarena]] 揭示现有 agent 在动态环境仅 39.6%；静态基准（GAIA/HLE）已饱和。
- 趋势：评测从"静态难题"转向"**持续演化 + 真实经济价值**"（呼应 huggingface 库的 Agents' Last Exam）——能否在不断变化的真实环境里**持续保持**能力，成为新标尺。

**方向 8️⃣：效率与成本——让自演化"用得起"**

- 现状：[[02-evocua]] 数万 sandbox rollout、[[14-quantaalpha]] 依赖 GPT-5.2、co-evolution 算力高——**门槛是普及的最大障碍**。
- 趋势：[[04-dreamgym]] 的"合成 warm-start + 少量真实 RL"、frozen model + 外部状态进化（[[09-skillsvote]]、SkillAdaptor）等**低成本路径**会持续受重视；training-free 演化是重要补充。

**方向 9️⃣：递归 / 元自改进（Recursive & Meta Self-Improvement）——从"改能力"到"改改进机制"**

- 现状：Hyperagents（Darwin Gödel Machine 思路）已指出"自我改进能力在 coding 域外难对齐"；Memento-Skills 让 agent 设计 agent；Need Sleep 让模型自我修改、巩固。
- 趋势：真正的递归自我改进——agent 不只解题、还改进自己的学习算法/元机制。这是通往"超级智能"叙事的核心，也是最难、最具风险的方向（[[21-agentic-reasoning-survey]] 列为深水区）。

**方向 🔟：开放世界 Bootstrap + 自演化的稳定性理论统一**

- 现状：OpenSkill/SCOPE 把自演化推到"无 verifier、无成功轨迹、只给 prompt"的开放世界；但 R-Few、[[08-ctx2skill]]、Guided Self-Evolving 都揭示**无引导自演化会塌缩**（concept drift / diversity collapse / mis-evolution）。
- 趋势：把"开放世界自造学习循环"与"防塌缩稳定性"统一——需要既能自举又不退化的机制。这是"自造数据"主线走到尽头后的必答题。

### 3.4 一句话总结现状与方向

> **现状**：2026 H1 的 agent evolve 已完成"从能不能进化 → 各流派范式确立 + 向多模态/空间/GUI 全面扩散 → 稳定可治理地进化 + 元进化萌芽"的三级跳；**skill 进化 + co-evolution + 防崩溃门控**是已成型的主流配方，**zero-data / open-world bootstrap**是最激进的前沿。
>
> **方向**：下一阶段的胜负手在 **① 终身跨域、② 可治理/可对齐、③ 进化单位上移到生态/元机制、④ 环境即服务、⑤ 理论化、⑥ world model 融合、⑦ 动态评测、⑧ 低成本、⑨ 递归/元自改进、⑩ 开放世界 bootstrap + 防塌缩**——谁能把"自演化"做得**又稳、又省、又可控、还能自我递归**，谁就能定义 2026 H2 乃至通向 superintelligence 的范式。
